# Sanitized Case Studies

These case studies are public-safe summaries of how community skill memory can
improve an agent research workflow. They do not publish alpha expressions, raw
forum exports, account data, platform screenshots, or investment advice.

这些案例只展示脱敏后的流程证据和指标信号，不公开 alpha 表达式、原始论坛内容、
账号信息、平台截图或投资建议。

## What Counts As Evidence

The purpose is to show that the skill system changes agent behavior:

- near-pass candidates are repaired before random regeneration;
- clone-like templates are blocked unless transformed;
- correlation failures are routed to settings isolation or family shift;
- pending checks stay in the check queue;
- submit review remains a human boundary.

## Case A: Submit10 Active Batch

Public-safe signal: one historical batch reached 10 ACTIVE outcomes after
crowded trunks were reduced and cross-domain overlays were added.

| Example | Domain | Sharpe | Fitness | Returns | Turnover | Self-corr note |
| --- | --- | ---: | ---: | ---: | ---: | --- |
| Case A1 | analyst, credit, microstructure | 1.53 | 1.29 | 8.91% | 0.1111 | 0.6666 |
| Case A2 | cashflow value, vwap reversal | 2.35 | 1.78 | 11.23% | 0.1967 | platform ACTIVE |
| Case A3 | cashflow, options overlay | 1.69 | 1.21 | 9.94% | 0.1924 | 0.6899 |

Distilled lesson: the useful change was not simple window tuning. The practical
repair pattern was to reduce an overcrowded main trunk and add a structurally
different overlay.

parameters and change the family.

## Case C: Submission Experience Rules

Public-safe signal: submission experience became field-level rule memory.

| Metric | Value |
| --- | ---: |
| Records | 6787 |
| Memory rows | 1791 |
| Rules | 131 |
| Artifacts scanned | 1615 |
| Success outcomes | 36 |
| Fail outcomes | 2924 |
| Pre-submit pass outcomes | 2996 |

Distilled lesson: repeated failures should become policy. For example,
concentrated-weight failures route to broad-overlay or source-design repair,
while pending checks stay out of submit review.
